# Contributing to OST Suite

Thank you for your interest in improving OST Suite! To keep the core system stable and secure, please follow these guidelines when submitting issues or pull requests (PRs).

## ⚠️ Main Rules

1. **No Hardware Interfacing Changes:** We **do not** accept Pull Requests that modify the core hardware interfacing code (e.g., sensor drivers, serial port logic). If you have a hardware integration request or bug fix, please open an Issue to discuss it first. We assume not everybody owns specific hardware used in this project.
   
2. **Review Your AI-Generated Code:**
   You are welcome to use AI tools to help write your code. However, you must **strictly review and test** all AI-generated code before submitting a PR. 

## How to Contribute

* **Bug Reports & Feature Requests:** Open an Issue and describe the problem or feature clearly. Include steps to reproduce bugs.
* **Pull Requests:** 
1. Fork the repo and create a new branch (`feature/your-feature-name`).
2. Ensure your code follows the existing style and architecture.
3. Test your changes locally.
4. Submit a PR with a clear description of what you changed and why.
